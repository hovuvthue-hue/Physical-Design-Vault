# GDS

## Definition
GDS (GDSII — Graphic Database System II) là binary file format chứa toàn bộ geometric description của IC layout ở tất cả các physical layers — từ các lớp bán dẫn bên dưới (N-Well, Diffusion, Poly) đến các lớp metal routing phía trên (M1 đến Mtop). GDS là representation đầy đủ và chi tiết nhất của design — không có abstraction, không có ẩn thông tin — và là file cuối cùng được gửi đến Foundry để chế tạo photolithography masks. Mỗi polygon trong GDS tương ứng với một vùng trên một mask layer cụ thể trong fabrication process. GDS là ranh giới chuyển giao thiết kế → sản xuất.

## Computed from
GDS được assembled qua quá trình hierarchical composition:

**Standard Cell GDS**: Do library vendor thiết kế thủ công (full custom layout) hoặc generate bởi P-cell tools. Mỗi Standard Cell có GDS riêng chứa đầy đủ transistor-level geometry — N-Well, P-diffusion, N-diffusion, Gate Poly, Contacts, Local Metal.

**Block-level GDS**: PnR tool (Cadence Innovus) export GDS sau khi Routing hoàn thành bằng cách: (1) instantiate từng Standard Cell GDS tại coordinates từ DEF Placement, (2) add routing geometry (wire rectangles và via arrays) từ DEF Route data, (3) merge tất cả thành một hierarchical GDS database.

**Chip-level GDS**: Hierarchically compose tất cả block-level GDS + Hard IP GDS + IO Pad GDS + I/O Ring + Seal Ring thành một file GDS duy nhất. File này sau đó được submit cho Foundry.

GDS chứa dữ liệu theo cấu trúc: LIBRARY → STRUCTURES (cells) → ELEMENTS (polygons, paths, references). Tất cả tọa độ được lưu dưới dạng integer với database unit thường là 1nm hoặc 0.001um.

## Constrains
- **[[Signoff]]**: GDS là input bắt buộc cho DRC (Design Rule Check — verify tất cả polygon geometries tuân thủ Foundry rules) và LVS (Layout vs. Schematic — verify GDS connectivity khớp với Netlist); GDS phải DRC-clean và LVS-clean trước Tape-out
- **[[ParasiticExtraction]]**: Signoff-level extraction dùng GDS thay vì DEF vì GDS chứa đầy đủ geometry kể cả các layers không có trong DEF (ví dụ: via arrays, local routing bên trong Standard Cells)
- **[[LEF]]**: Macro LEF được generate từ Standard Cell GDS qua Abstract Generator — tool này strip toàn bộ layers không cần thiết cho PnR và extract chỉ metal layers, Pin locations, và OBS regions

## Requires
- [[Routing]] — Route DB (DEF với full wire geometry) là primary source để generate block-level GDS; sau khi Routing hoàn thành, PnR tool stream out GDS bằng cách combine placement coordinates với Standard Cell GDS templates
- [[DEF]] — Post-route DEF chứa cell placement coordinates và routing wire geometry; GDS export step đọc DEF để biết tọa độ của từng instance và wire segment

## Used by
- [[Signoff]] — DRC tool (Calibre, IC Validator) đọc GDS + DRC rule deck từ Foundry để verify mọi polygon; LVS tool (Calibre LVS) extract netlist từ GDS và compare với Verilog Netlist; GDS-clean signoff là hard prerequisite cho Tape-out
- [[ParasiticExtraction]] — Signoff-level extraction (QRC, StarRC) đọc GDS để có full geometry kể cả intra-cell routing; cho kết quả chính xác hơn so với extraction từ DEF-only
- [[LEF]] generation — Abstract Generator đọc Standard Cell GDS để tạo Macro LEF; process này xảy ra một lần khi library được developed, không phải trong PD flow

## Key insight
[USER REVIEW — draft suggestion]: GDS là "bản vẽ xây dựng" của chip — mỗi polygon trong file này tương ứng với một vùng vật liệu cụ thể sẽ được lắng đọng, khắc, hoặc implant trên silicon wafer. Đây là lý do GDS là format duy nhất không có "abstraction" hay "approximation" — tất cả phải chính xác đến nanometer. Điểm quan trọng về Tape-out: Foundry charge "mask cost" theo số lượng mask layers — một chip advanced node như TSMC 3nm có thể cần 80+ mask layers, mỗi mask trị giá hàng triệu USD. Một lỗi DRC trong GDS (dù chỉ một polygon sai) có thể dẫn đến fabrication defects và toàn bộ wafer lot bị lỗi — đây là lý do DRC signoff được kiểm tra cực kỳ nghiêm ngặt và thường require zero violations trước Tape-out. OASIS là successor format của GDSII với compression tốt hơn — cùng data nhưng file size nhỏ hơn 5–10 lần, đang được adopt tại các advanced nodes vì GDS file size có thể lên đến hàng trăm GB.
